Introducció

Els llenguatges de programació orientada a objectes són dels més utilitzats: Python, JavaScript i C++. En canvi, altres llenguatges com C no estan orientats a objectes.

Advantatges de la POO

  1. Encapsulació: Consisteix en agrupar dades (propietats) i les funcions s'acostumen a anomenar mètodes que operen sobre les dades anteriors en una única entitat anomenada classe. Té els següents avantatges: la classe queda oculta i només podem interactuar amb l'objecte creat a partir de la classe. És més difícil cometre errors en crear objectes perquè la classe normalment es defineix bé. El codi és més segur perquè només interactuem amb objectes i no amb classes.
  2. Reutilització del codi: podem utilitzar la mateixa classe o el mateix objecte en diferents parts del programa o en programes diferents. Els avantatges són: Evita duplicacions del codi. Podem crear moltes instàncies diferents de la mateixa classe o objectes amb facilitats. També puc fer una classe complexa en forma de biblioteca constituïda per una o més classes i utilitzar-la en molts programes diferents.
  3. Herència: Ens permet crear una classe o subclasse heretant les propietats i els mètodes d'una classe existent o d'una superclasse. Té l'avantatge d'extendre les propietats i els mètodes d'una superclasse o classe a una subclasse o classe. La nomenclatura pot ser jeràrquica superclasse per sobre de subclasse o heretada: Classe principal i classe heretada de la classe principal. L'avantatge és que evitem escriure codi redundant i no repetir un codi quan volem o variar les funcions d'una classe.
  4. Polimorfisme: Permet crear objectes diferents a partir de la mateixa classe, per exemple variant els arguments. L'avantatge és que puc crear objectes diferents variant els arguments o cridant diferents mètodes de la classe a cada objecte.
  5. Mantenibilitat: El codi creat per classes és més fàcil de mantenir perquè cada classe té una responsabilitat específica. Si tenim el codi separat en classes, podem trobar els errors amb més facilitat.
  6. Escalabilitat: El projecte és més escalable, vol dir que es pot fer més gran i complex perquè podem afegir noves funcionalitats sense afectar al codi existent.

  8. Abstracció: Permet fer més clar el codi perquè oculta els detalls interns de les complexitats.
  9. Organització: Permet tenir el codi més ordenat i més llegible per altres programadors facilitant el manteniment a llarg termini i la documentació

Programació orientada a objectes

Llenguatges de programació com C++, Javascrypt o Python sôn POO, en canvi, C, no es orientada a objectes

En el exemple que veurem, construirem una classe "Mosca", les classes acostumen a tenir noms en Majùscula, i, dintre seu, hi ha diferents parts.

La primera part, utilitzem la paraula clau "constructor" per formar la propia classe, que actua com a motlle. Dintre del constructor, introduim arguments que afecten a la propia classe designats amb la paraula "this".

Al principi donem valors inicals als parametres/propitats de la classe. Analitzarem la classe mosca:


      class Mosca { //L'objectiu de crear una classe es poder crear diverses instÂncies de objectes d'una mateixa classe. Les Classes ens ajuden a tenir el codi ordenat i encapsulat
      constructor(x, y) {
        this.x = x;
        this.y = y;
        // La mida base determina la grandària de la mosca
        this.mida = random(10, 20);
        this.velocitat = random(1, 3);
        // Offsets per al moviment suau amb noise
        this.offsetX = random(1000);
        this.offsetY = random(1000);
      }
